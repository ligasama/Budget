# Budget v1 README：预算驱动的快速记账工具

## 产品定位

`Budget` 不是普通流水账，也不是复杂理财平台。它的核心定位是：

**先做预算，再记录消费，用每一笔流水实时反馈本月预算执行情况。**

用户第一次或每月月初进入时，需要先看到“这个月的钱被怎么安排”；用户日常打开时，最高频动作是立刻记一笔支出或额外收入。预算页应该是动态 dashboard，不是静态配置页。

## v1 核心目标

1. 首屏就是本月预算 dashboard，让用户一眼知道本月预算、收入、支出、结余和四个账户状态。
2. 底部中间提供一个突出的 `+` 核心按钮，专门用于快速记账，区别于其他静态页面入口。
3. 每记录一笔消费或额外收入，首屏 dashboard 立即变化，让用户感到“预算正在被实时消耗或补充”。
4. 保留现有四账户预算体系，但把它从编辑表格升级成可读、可点、可反馈的预算执行面板。
5. 增加年度宏观视图，让用户能看全年收入、支出、结余、月度变化和分类排行。

## 信息架构

底部导航建议采用竞品式结构，但 `+` 是核心动作，不是普通页面：

```text
预算        明细        +        图表        我的
Dashboard   流水列表    快速记账   月/年分析   设置/数据
```

### 预算

默认首页，也是本月 dashboard。

展示本月预算账户、收入支出、结余、预算执行进度，以及下方四个账户预算：

- 生存账户
- 长期资产账户
- 生活升级账户
- 自由账户

### 明细

展示所有流水，按日期分组。

用于查账、删除流水、后续编辑流水。

### `+` 记账

底部中间凸出的主按钮。

点击后打开快速记账面板，支持支出、收入、分类宫格、计算器输入、备注、日期和账户选择。

### 图表

展示月度和年度分析。

包括趋势、分类排行、年度账单和异常月份。

### 我的

放低频能力。

例如账户管理、分类管理、数据导出、清空数据、主题设置。v1 可以只做最小设置页。

## 首屏预算 Dashboard

预算页是用户点进来的首页。它不是“预算编辑页”，而是一个会跟随流水变化的预算执行仪表盘。

### 顶部总览卡片

展示本月核心数字：

- 本月预算
- 本月收入
- 本月支出
- 本月结余
- 预算剩余

推荐主数字优先显示“预算剩余”或“本月结余”，因为它直接回答用户最关心的问题：这个月还能不能花。

示例：

```text
6月预算剩余
¥8,420

收入 ¥18,000    支出 ¥4,580
预算 ¥13,000    结余 ¥13,420
```

### 动态反馈

每新增一笔支出：

- 本月支出增加。
- 本月结余减少。
- 对应账户和分类的剩余额度减少。
- 如果接近预算上限，卡片状态变成提醒态。
- 如果超预算，卡片状态变成警示态。

每新增一笔额外收入：

- 本月收入增加。
- 本月结余增加。
- 出现“待分配收入”提示，引导用户把额外收入分配到四个账户。

额外收入不要默认悄悄混进预算，否则用户会失去“先预算再消费”的控制感。

### 四账户预算区

下方展示四个账户，每个账户是一张预算执行卡：

- 账户名称
- 本月预算
- 已用金额
- 剩余金额
- 使用进度
- 账户下 Top 分类

四账户建议：

1. 生存账户：房租、餐饮、交通、通讯、日用品、健康。
2. 长期资产账户：指数基金、债基、货币基金、应急金。
3. 生活升级账户：家居、电子产品、旅行、身体和心情改善。
4. 自由账户：临时消费、娱乐、小额冲动消费。

点击账户卡片进入该账户详情，展示账户内分类预算和流水。

### 状态提示

Dashboard 需要根据数据给出轻量提示：

- “本月自由账户还剩 ¥320，今天适合收一收。”
- “餐饮已用 82%，还剩 ¥260。”
- “新增收入 ¥1,200 待分配。”
- “本月结余为负，优先检查自由账户和生活升级账户。”

提示应该来自真实数据，而不是静态说明文案。

## `+` 快速记账按钮

`+` 是 v1 的核心交互，需要明显区别于其他导航项。

### 视觉规则

- 放在底部导航中间。
- 使用圆形或半悬浮按钮。
- 尺寸大于普通 tab 图标。
- 使用品牌强调色。
- 文案可以是 `记账`，图标是 `+`。
- 点击后不切换普通页面，而是打开快速记账面板。

### 打开方式

移动端：

- 点击 `+` 后，从底部弹出全屏或半屏记账面板。
- 面板顶部保留关闭按钮。
- 完成记账后自动收起，回到预算 dashboard，并刷新数字。

桌面端：

- 点击 `+` 后，打开居中的记账面板或右侧抽屉。
- 背景 dashboard 仍可见。

## 快速记账面板

目标是让用户 3-10 秒完成一笔账。

### 顶部

- 支出 / 收入 分段控制。
- 日期，默认今天。
- 账户，默认上一次使用账户。

### 中部

分类宫格：

- 每行 4 个分类。
- 图标 + 名称。
- 常用分类优先。
- 支出和收入使用不同分类集合。

支出分类示例：

- 餐饮
- 购物
- 日用
- 交通
- 蔬菜
- 水果
- 零食
- 运动
- 娱乐
- 通讯
- 服饰
- 美容
- 住房
- 居家
- 孩子
- 长辈

收入分类示例：

- 工资
- 奖金
- 副业
- 理财
- 报销
- 红包
- 退款
- 其他

### 底部

- 金额显示区。
- 备注输入。
- 图片或标签入口可先预留。
- 计算器键盘：0-9、小数点、退格、加、减、今天、完成。

### 交互规则

- 默认是支出。
- 默认选中最近一次使用的支出分类。
- 支持 `21+35` 这样的简单加减表达式，完成时保存为 `56`。
- 完成后清空金额和备注，但保留类型、分类和账户，方便连续记账。
- 记账成功后出现轻反馈，例如“已记入餐饮，预算还剩 ¥260”。
- 完成后 dashboard 的总览卡片和四账户卡片同步刷新。

## 预算和记账的关系

v1 必须把“预算”和“流水”绑定起来。

### 支出流水

一笔支出应该影响：

- 本月支出
- 本月结余
- 对应账户已用金额
- 对应账户剩余预算
- 对应分类已用金额
- 年度支出统计

### 收入流水

一笔收入应该影响：

- 本月收入
- 本月结余
- 年度收入统计

如果是工资收入，可以进入本月预算分配；如果是额外收入，先进入“待分配收入”。

### 待分配收入

当用户记录额外收入时，预算 dashboard 顶部出现提示：

```text
有 ¥1,200 待分配
分配到：生存 / 长期资产 / 生活升级 / 自由
```

v1 可以提供一个简单操作：

- 全部分配到自由账户。
- 按当前预算比例分配。
- 手动分配。

## 月度明细页

明细页承接查账，不承担最高频记账动作。

### 列表规则

- 按日期分组。
- 每天显示收入、支出、结余。
- 每笔流水显示分类、备注、账户、金额。
- 支出用负向视觉，收入用正向视觉。
- 支持删除。
- v1 可暂不支持编辑，编辑放到 v1.1。

### 筛选

v1 至少支持：

- 按月份筛选。
- 按支出 / 收入筛选。
- 按账户筛选。

## 图表页

图表页分为月度和年度。

### 月度图表

展示：

- 本月支出趋势。
- 分类支出排行。
- 四账户预算使用进度。
- 最大几笔支出。

### 年度图表

展示：

- 年度收入。
- 年度支出。
- 年度结余。
- 月均支出。
- 储蓄率：`年度结余 / 年度收入`。

如果没有收入，储蓄率显示为 `--`。

### 年度账单

按 12 个月展示：

- 月份
- 收入
- 支出
- 结余
- 储蓄率
- 异常状态

异常规则 v1：

- 支出高于全年月均支出 150%，标记为“高支出”。
- 结余为负，标记为“入不敷出”。
- 收入为 0 且支出大于 0，标记为“无收入支出”。

点击某个月进入该月明细。

## 数据模型调整

当前交易结构需要支持收入、支出、账户、预算扣减和创建时间。

```js
{
  id: "tx_xxx",
  type: "expense" | "income",
  date: "2026-06-16",
  accountId: "survival",
  categoryId: "food",
  detailId: "mealDelivery",
  amount: 56,
  note: "早餐+午餐",
  source: "manual",
  createdAt: 1780000000000
}
```

建议增加预算月对象：

```js
{
  month: "2026-06",
  plannedIncome: 18000,
  pendingIncome: 1200,
  accounts: {
    survival: { budget: 6200 },
    assets: { budget: 6800 },
    upgrade: { budget: 3000 },
    freedom: { budget: 2000 }
  }
}
```

统计值不要长期手写保存，优先由预算和流水计算得到，避免数据不一致。

## 本地存储

建议升级 storage key：

- 当前：`cashflow-layering-state-v4`
- v1：`budget-state-v1`

迁移策略：

1. 启动时优先读取 `budget-state-v1`。
2. 如果不存在，读取旧 key。
3. 迁移旧账户和旧流水。
4. 没有 `type` 的旧流水默认视为 `expense`。
5. 没有 `accountId` 的旧流水按旧分类映射到账户。
6. 保存到新 key。
7. 不删除旧 key，避免回滚时数据丢失。

## v1 验收标准

### 预算 Dashboard

- 用户打开应用后，默认看到本月预算执行情况。
- 用户能看到本月预算、收入、支出、结余和预算剩余。
- 用户能看到四个账户分别还剩多少钱。
- 新增支出后，对应账户剩余预算立即减少。
- 新增额外收入后，出现待分配收入提示。

### 快速记账

- 用户可以通过底部中间 `+` 在 10 秒内完成一笔支出。
- 用户可以记录收入。
- 用户可以连续记录多笔流水，不需要反复选择相同分类。
- 完成后自动回到预算 dashboard。
- 完成后 dashboard 数字立即更新。

### 明细

- 流水按日期分组。
- 每天有收入、支出、结余小计。
- 删除流水后，dashboard 和图表同步更新。

### 年度

- 用户能看到全年收入、支出、结余和月均支出。
- 用户能看到 12 个月的收入、支出、结余。
- 用户能识别高支出月份和负结余月份。
- 用户能看到全年支出最多的分类。

## v1 暂不做

- 多设备同步。
- 登录和账户体系。
- 银行账单导入。
- OCR 识别小票。
- 复杂资产负债管理。
- 借贷、人情、报销流转。
- 多币种。
- 云端备份。

## 最终判断标准

`Budget` v1 成功的标准不是功能多，而是闭环清楚：

1. 月初或工资到账后，用户在预算 dashboard 安排本月的钱。
2. 日常消费时，用户点击底部 `+` 快速记一笔。
3. 每一笔流水都会让 dashboard 发生变化。
4. 用户随时知道这个月还能花多少，年底知道钱去了哪里。
